# balance-eval-guide — 工具數值解讀指南 + BR 公式校正

## 問題摘要

| 項目 | 說明 |
|------|------|
| 現象 | balance-eval.html 輸出的 WTS / TDG / BR / PThreat 數字不知道怎麼看 |
| 根因 A | BR 的閾值（0.8 / 1.5 / 4.0）是拍腦袋設的，實際遊戲數值全在 0.07–0.30 之間，全部標紅「過難」沒有意義 |
| 根因 B | BR 公式的單位是「秒⁻¹」（每秒消耗幾分之幾波次血量），缺少路徑時間因子，不直觀 |
| 解法 | ① 本文件解釋各指標的真實含義 ② step1 修正 BR 公式加入路徑時間，讓輸出有意義 |

## 各指標的真實含義

### WTS（波次威脅分數）— 單位：EHP 總量

```
WTS = count × (HP÷(1−armor)) × skill乘數
```

**白話：「這波如果要打完，需要造成多少有效傷害。」**

| 代表波 | WTS | 說明 |
|--------|-----|------|
| W1 普通小兵 | ~440 | 基準線 |
| W4 鐵甲衛士 | ~1227 | W1 的 2.8× |
| W8 熔岩巨像 | ~3286 | W1 的 7.5× |
| W12 深淵領主 | ~4672 | W1 的 10.6× |
| W16 鋼鐵巨獸 | ~8771 | W1 的 20× |
| W20 終焉之王 | ~16445 | W1 的 37× |

**怎麼看：** 關注 WTS△（本波÷上波），若 > 3.0 就是難度 spike，設計上應在 spike 前給玩家足夠資源升塔。

---

### Gold Flow（可用金幣）— 單位：金幣 g

```
available = 起始230g + 累積擊殺金 + baseIncome × 波數
```

**白話：「如果玩家完全沒花錢，到這波手上最多有多少錢。」**

這是「理論上限」。實際上玩家要建塔，真正可花的錢大約是上限的 30–50%。

| 代表波 | 理論上限 | 實際估算（×0.4） |
|--------|---------|----------------|
| W4 | 494g | ~198g |
| W8 | 930g | ~372g |
| W12 | 1446g | ~578g |
| W20 | 2972g | ~1189g |

**怎麼看：** 用來估算「這波玩家應該已經有多少塔了」。金幣增速 vs WTS 增速的相對趨勢才重要。

---

### TDG（塔的 DPS/Gold 效率）— 單位：DPS per gold

```
TDG = damage × atkSpd × AOE倍率 × score_adj ÷ 累積成本
```

**白話：「每塊錢能買到多少 DPS。」**

| 代表塔 | DPS(adj) | 累積金 | TDG |
|--------|----------|--------|-----|
| Lv2 箭塔 | 57 | 130g | **0.44** ← Lv1-2 最佳 |
| Lv3 焰砲台（AOE） | 166 | 260g | **0.64** ← Lv3 最佳 |
| Lv4 磐石（純土） | 155 | 510g | **0.30** |
| Lv5 火山塔（AOE） | 220 | 910g | **0.24** |
| Lv6 磐石終極 | 224 | 1460g | **0.15** |

**重要規律：TDG 隨等級「下降」！高等級塔不是更「划算」，而是 DPS 絕對值更高。**
早期塔（Lv2-3）的 TDG 反而最高，是因為投資成本低。

**怎麼看：** 比較同等級的塔，找性價比最高的投資目標。跨等級比較要用「需要打多久才值回票價」的邏輯。

---

### BR（Balance Ratio）— 原公式有問題，需修正

**原公式：** `BR = TPI / WTS = (available_gold × bestTDG) / WTS`

原公式的單位是「**1/秒**」（每秒消耗幾分之幾波次血量），數值全在 0.07–0.30。

設定 0.8 / 1.5 / 4.0 的閾值根本沒意義（全部標紅），需要修正。

**修正後公式：**

```
TTK（擊殺時間）= WTS / TPI = WTS / (gold × bestTDG)
BR_fix = path_time / TTK = path_time × TPI / WTS
       = path_time × gold × bestTDG / WTS
```

BR_fix 代表：**「在路徑內，玩家最佳投資能清掉這波的幾倍？」**

- `BR_fix < 0.4`：利用率 50% 的情況下無法在路徑內清除 → 🔴 設計有問題
- `BR_fix 0.4–0.8`：緊繃，需要高效率建塔 → 🟡 挑戰
- `BR_fix 0.8–2.5`：正常，有餘裕 → 🟢 理想
- `BR_fix > 2.5`：過於輕鬆 → ⬜ boring

**用實際數值驗算（path_time=15s）：**

| 代表波 | WTS | TPI(W×TDG) | BR_fix | 判定 |
|--------|-----|-----------|--------|------|
| W1 | 440 | 230×0.44=101 | 15×101/440 = **3.4** | ⬜輕鬆 |
| W4 Boss | 1227 | 494×0.64=316 | 15×316/1227 = **3.9** | ⬜輕鬆 |
| W8 Boss | 3286 | 930×0.30=279 | 15×279/3286 = **1.3** | 🟢理想 |
| W12 Boss | 4672 | 1446×0.24=347 | 15×347/4672 = **1.1** | 🟢理想 |
| W16 Boss | 8771 | 2078×0.20=416 | 15×416/8771 = **0.71** | 🟡挑戰 |
| W20 Boss | 16445 | 2972×0.15=446 | 15×446/16445 = **0.41** | 🔴過難 |

→ W1-4 過於輕鬆（早期應有挑戰），W16+ 開始緊繃，W20 Boss 理論上偏難。

### PThreat（送兵威脅值）

```
PThreat = EHP_送兵 / (對手DPS × path_time)
```

**白話：「你送的兵，對手需要幾條完整路徑才能清掉。」**

- `PThreat < 0.5`：對手很輕鬆清 → 送兵幾乎沒威脅
- `PThreat 0.5–1.0`：對手要分心應對 → 有一定壓力
- `PThreat > 1.0`：對手「一條路清不完」→ 送兵一定會造成基地傷害

**注意：** 這是「單塔應對」的保守估算。現實中對手有多塔，PThreat 要 > 1.5–2.0 才真的危險。
