# multishot-analysis：多重箭 vs chain vs multishot 設計分析

## 一、三種現有多目標技能的完整機制

### chain（連鎖）
```js
// game.js ~2901
chainTargets = targets.filter(e => e !== target).slice(0, chainSk.targets)
chainDmg[i] = floor(effDmg × decay^(i+1))
```
- **觸發**：每次攻擊，命中主目標後自動跳躍
- **目標選取**：從已在射程內的 `targets`（pathIdx 排序）取前 N 個，排除主目標
- **傷害**：每跳 ×decay（預設 0.7），第一跳 70%，第二跳 49%
- **依賴**：射程內需有多個敵人才有效果

### multishot（三連射）
```js
// game.js ~2873
if (multiSk && tw.atkCount % multiSk.every === 0) shots = multiSk.shots
// 每 shots 中：s=0 打主目標，s=1..n 打 targets[s]（其他射程內敵人）
// 擊殺後：_killRushBonus = killBonus，_killRushTimer = killDur
```
- **觸發**：每 `every` 次攻擊觸發一次，其餘攻擊是普通單體
- **目標選取**：同一次攻擊同時打 `shots` 個不同敵人（射程內，pathIdx 排序取前 N）
- **傷害**：每發都是完整 effDmg（不衰減）
- **額外效果**：擊殺後 killRush（預設 +50% 攻速，持續 3s）
- **節奏感**：有明顯的「普通 → 普通 → 爆發」節奏

### pierce（穿透，已改版）
```js
// 直線篩選（垂直距離 ≤ 0.6 格），依距離排序
// ratio = max(0.3, 1 − i × dmgDown)
```
- **觸發**：每次攻擊，命中直線上所有敵人
- **目標選取**：需在直線上（有方向限制）
- **傷害**：主目標 100%，後方逐漸遞減，下限 30%

## 二、三者對比表

| 維度 | chain | multishot | pierce |
|------|-------|-----------|--------|
| 觸發頻率 | 每次攻擊 | 每 N 次攻擊 | 每次攻擊 |
| 目標限制 | 射程圓圈內（任意） | 射程圓圈內（任意） | 直線方向上 |
| 傷害模式 | 主目標 100% + 跳躍衰減 | 全部 100%（不衰減）| 主目標 100% → 遞減至 30% |
| 目標選取邏輯 | 跳躍（鏈式，可控方向感弱）| 獨立各打一個 | 固定直線方向 |
| 額外效果 | 無 | killRush（擊殺加速）| 無 |
| 強項場景 | 敵人密集時補傷 | 多路或清多目標 | 敵人排成一列 |
| 弱項場景 | 稀疏波次 | 單一高 HP 目標 | 敵人非同向分散 |

## 三、「多重箭」的設計空間

### 現有三種技能的共同局限

| 限制 | chain | multishot | pierce |
|------|-------|-----------|--------|
| 需要特殊時機/條件 | 射程內需多敵 | 每 N 次才爆發 | 敵人需在直線上 |
| 穩定多目標輸出？ | 條件依賴 | 非穩定（有節奏）| 方向依賴 |

**設計空白**：沒有一種技能能「每次攻擊穩定地同時對多個目標輸出，不需要方向對齊，也不需要等待觸發節奏」。

### 多重箭的可能定位

**定義**：每次攻擊同時發射 `shots` 支箭，分別打射程內的不同目標，每支箭傷害為 `effDmg × ratio`。

```
// 概念程式碼
multiArrow: { shots: 2, ratio: 0.6 }
// 每次攻擊：打主目標（effDmg×ratio）+ 額外 shots-1 個目標（各 effDmg×ratio）
// 或：主目標打 effDmg × 1.0，額外目標打 effDmg × ratio
```

### 多重箭 vs 三者差異

| 維度 | 多重箭（提案）| chain | multishot | pierce |
|------|------------|-------|-----------|--------|
| 觸發頻率 | **每次攻擊** | 每次攻擊 | 每 N 次 | 每次攻擊 |
| 傷害穩定性 | **高（無節奏）**| 中（需多敵）| 低（節奏型）| 中（需直線）|
| 衰減？ | **有**（ratio < 1）| 有（decay）| **無** | 有（falloff）|
| 方向限制？ | **無** | 無 | 無 | **有**（直線）|
| 額外效果 | **無** | 無 | killRush | 無 |
| DPS 上限 | 正比敵人數 | 射程內敵人數 | 固定爆發 | 直線敵人數 |

## 四、是否值得加入？

### 支持加入的理由

1. **填補設計空白**：穩定、無條件、無方向的多目標技能，現有三種都無法提供
2. **與弓手基底吻合**：多重箭是弓手塔的直覺技能，narrative 一致
3. **差異化明確**：與 multishot（節奏型爆發）、chain（跳躍衰減）、pierce（直線貫穿）各有明顯不同

### 需要謹慎的地方

1. **DPS 計算複雜**：若每支箭都附帶 proc 技能（chill/shred），多重箭等於多重 proc → 在 procMinInterval 未實作的情況下可能過強
2. **與 multishot 的體感重疊**：兩者都是「同時打多個目標」，玩家可能難以感受差異；需要拉大設計距離
   - 建議：multishot 維持爆發+擊殺加速的「風系獵手」身份；多重箭定位為穩定清場型，不帶觸發效果
3. **scoreBase 難以估算**：DPS 收益正比敵人密度，技能分難以在稀疏/密集波次間保持一致

### 建議設計方向（若決定加入）

```
multiArrow: {
  shots: 2,      // 同時發射的箭支數（含主目標）
  ratio: 0.65,   // 每支箭的傷害係數（主目標也折損，或只折損附加箭）
  scoreBase: 35,
}
```

**兩種參數方案**：
- A. 主目標也折損（`ratio` 套用到全部箭）：`shots=2, ratio=0.65` → 總傷害 = 1.3× effDmg 分攤給 2 人
- B. 主目標不折損（`ratio` 只套用到額外箭）：`shots=2, ratio=0.65` → 主目標 100% + 副目標 65%

方案 B 更符合直覺（主要還是打主目標），方案 A 更平衡（避免免費 AOE）。

## 五、結論與決策問題

**多重箭≠連鎖、多重箭≠三連射**——三者核心差異：
- chain：「跳躍，傷害逐漸衰減」
- multishot：「節奏爆發，每發滿傷，擊殺加速」
- 多重箭（提案）：「每次攻擊穩定分射，固定傷害折損」

### 需使用者確認

1. **是否加入多重箭技能？**
2. **若加入，採用方案 A（全部折損）還是方案 B（主目標不折損）？**
3. **多重箭應屬哪個元素身份？**（目前考慮：無元素 / 風元素「精準多目標」身份 / 獨立於元素）
4. **是否需要先實作 procMinInterval，再加多重箭？**（防止多重箭+proc技能的疊加問題）
