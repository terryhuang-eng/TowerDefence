# 概念細化分析：土的穿透 + 風的 ramp 保留機制

---

## 一、土：單體大傷害 + pierce 技能

### 新概念
每次攻擊只打一個主要目標（單體），但攻擊穿透——貫穿射程內所有敵人，後排越痛。

### 為什麼這比「AOE + pierce」更好

**原本的「大範圍傷害 + 穿透」問題：**
兩個技能描述的是「打到多個目標」，重複了，概念上有歧義。

**新概念的優點：**

| 層面 | 分析 |
|------|------|
| 概念清晰 | 土 = 一發重型砲彈貫穿整列敵人，後排越痛，不是「爆炸波及」|
| 「大範圍感」的來源 | range 數值大 + pierce 打到很多目標，玩家感受是「一發打穿整條路」|
| 無技能衝突 | 只有 pierce 一個技能，不需要把 AOE 和 pierce 硬合在一起 |
| 高密度獎勵 | 敵人排成一列 = 後排更痛 = 有策略思考空間（放在轉角前）|

### 土與其他攻擊類型的對比

| 元素 | 攻擊類型 | 多目標方式 | 傷害分布 |
|------|---------|-----------|---------|
| 🔥 火 | 單體 | 無（全力集中一個）| 全額 + DOT 累積 |
| 💧 水 | 範圍（AOE）| 圓形爆炸/波及 | 均等分布 |
| ⛰️ 土 | 穿透（pierce）| 貫穿線型 | 後排遞增（越打越痛）|
| 🌪️ 風 | 單體 | 無（快速逐一）| 全額 × ramp 倍率 |
| ⚡ 雷 | 多發（multishot）| 同時多個目標各一發 | 各自全額 |

三種打多目標的方式：**AOE（水）= 覆蓋、pierce（土）= 貫穿、multishot（雷）= 齊射**，各有語意，互不重疊。✅

### 概念可行性

| 問題 | 分析 |
|------|------|
| 普通波次能殺嗎？ | pierce 打穿整列，後排遞增 → 密集敵群時總傷害高 ✅ |
| 稀疏敵人怎麼辦？ | 只打一個目標時 = 高單體傷害 + 大射程補償 ✅ |
| 攻速慢的問題？ | 大射程讓敵人在攻擊範圍內更久，補償攻速慢 ✅ |

**結論：✅ 可行，且比原本「AOE + pierce 雙技能」更清晰。**

土的定位：**「重型穿甲砲 — 一發貫穿整排，後排越痛，適合在通道前守株待兔」**

## 二、風：ramp 不完全歸零（保留機制）

### 問題回顧
原本 ramp = 攻擊同一目標每次 +攻速，切換目標立即歸零。
TD 實戰中敵人快速死亡 → 頻繁換目標 → ramp 永遠低 → 風塔偏弱。

### 新概念：切換目標時保留部分 ramp

有兩種設計方向：

---

### 方向 A：固定保留百分比

切換目標時，ramp 保留 X%（例如 50%）。

```
例：ramp 疊到 +0.8 攻速
→ 換目標後剩 +0.4
→ 繼續打新目標從 +0.4 開始疊
```

| 優點 | 缺點 |
|------|------|
| 規則簡單，玩家可預期 | 不夠「風」的感覺（機械式百分比）|
| 普通波次不會從零開始 | 保留比例需調參（50%？70%？）|

---

### 方向 B：動態衰減（時間型）

切換目標後，ramp 不立即歸零，而是**隨時間緩慢衰減**。
若快速換到下一個目標，幾乎不損失；停止攻擊幾秒後，才歸零。

```
例：ramp +0.8，切換目標
→ 0.5 秒後仍有 +0.6（少量損失）
→ 2 秒不攻擊後降至 +0.2
→ 5 秒後歸零
```

| 優點 | 缺點 |
|------|------|
| 「風的慣性/動量」感 — 符合風的物理直覺 | 計時器邏輯稍複雜 |
| 快速清群時 ramp 幾乎不損失（敵人一個接一個死）| 玩家不容易看出「現在剩多少 ramp」|
| 停止輸出才有懲罰（不是換目標就懲罰）| |

---

### 方向 C：對每個目標獨立記憶（per-target ramp）

風記住攻擊過每個敵人的 ramp 層數。打 A 打到 +0.5，轉去打 B，回到 A 時仍有 +0.5。

```
敵人 A：ramp = +0.5（打了 10 次）
敵人 B：ramp = +0.3（打了 6 次）
敵人 C：ramp = 0（新目標）
```

| 優點 | 缺點 |
|------|------|
| Boss 反覆切換也不會損失 | 實作複雜（每個敵人需要獨立計數）|
| 策略深度：風擅長「重點目標」| 資料需存在敵人身上（per-enemy）|

---

### 推薦：方向 B（動態衰減）

**理由：**
1. **語意最自然**：風不是瞬間靜止的，它有動量，慢慢消散
2. **普通波次友善**：密集敵群快速死亡 → 攻擊間隔短 → ramp 幾乎不衰減 → 風在清群時能維持高攻速
3. **停頓有懲罰**：如果你讓風塔空置（沒有目標），動量消散，這是合理的代價
4. **實作可行**：只需要一個浮點計時器（_rampDecayTimer），不需要 per-enemy 資料

**衰減參數方向：**
- 每秒衰減 X 層（例如每秒 -2 層 ramp）
- 找到目標後立即停止衰減，繼續累積
- 這讓「快速清群流」的風幾乎沒有損失，「目標稀疏時」有懲罰

| ⚡ 雷 | 快 | 單體+三連射齊發 | 同時多目標 | 射速光環 |
| ⬜ 無 | 中 | ??? | ??? | 擊殺獎金 |

**多目標覆蓋分工（確認無重疊）：**
- 圓形覆蓋 = 💧 水
- 線型貫穿 = ⛰️ 土
- 齊射多目標 = ⚡ 雷
- 純單體 = 🔥 火、🌪️ 風、⬜ 無
