# 概念細化 7：無屬性 — 雙版財富積累 + 利息完整設計

## 確認的三個機制

| 機制 | 基準 | 效果 | 取捨 |
|------|------|------|------|
| 財富積累（當前持有）| currentGold | 傷害 × f(currentGold) | 花錢升塔 = 傷害暫降，取捨感強 |
| 財富積累（累積收益）| totalGoldEarned | 傷害 × g(totalEarned) | 只增不減，獎勵積極 LTW 送兵 |
| 利息 | currentGold | 波末直接加 gold | 不是 income，不是永久 |

---

## 一、財富積累（當前持有）

```
damage_bonus_A = floor(currentGold / divisorA)
```

**行為：**
- 持有 200g → +10 傷害（divisorA=20）
- 升塔花 250g → 金幣剩 50g → 傷害暫時 +2
- 幾波後金幣回收到 300g → 傷害 +15

**設計張力：**
- 「把錢花光就少輸出，存太多金幣不升塔又會被怪打穿」
- 不存在「最優解」，每個玩家的節奏不同都有代價

**與 killGold 的協同：**
killGold 讓金幣更容易維持在高位，間接保住傷害加成。

---

## 二、財富積累（累積收益）

```
damage_bonus_B = floor(totalGoldEarned / divisorB)
```

**totalGoldEarned 的定義：**
本局從所有來源累積賺到的總金幣（包含 income、killGold、利息、波次獎勵），只增不減。

**行為：**
- W5 時已賺過 800g → +8 傷害（divisorB=100）
- W15 時已賺過 3000g → +30 傷害
- 送兵叫兵 = income 提升 = 後續每波賺更多 = totalEarned 更快增加

**與 LTW 系統的協同：**
積極送兵的玩家 income 更高 → 每波賺更多 → totalEarned 增加更快 → 此塔後期更強。
純防守（不送兵）的玩家 income 低 → totalEarned 慢 → 此塔後期較弱。

→ 「無」元素鼓勵積極使用 LTW 系統。

---

## 三、兩種傷害加成的疊加方式

**合併公式：**
```
effectiveDmg = baseDmg + damage_bonus_A + damage_bonus_B
```

加法疊加（不是乘法），避免指數爆炸。

**數值範例（參考值，需比對其他塔）：**

| 時機 | currentGold | totalEarned | A加成 | B加成 | baseDmg | 有效傷害 |
|------|------------|------------|-------|-------|---------|---------|
| W3 剛升LV5 | 80g | 600g | +4 | +6 | 55 | 65 |
| W8 持金充足 | 300g | 1500g | +15 | +15 | 55 | 85 |
| W8 剛花錢升塔 | 30g | 1500g | +1 | +15 | 55 | 71 |
| W15 LTW積極 | 200g | 4000g | +10 | +40 | 75 | 125 |
| W15 純防守 | 100g | 2000g | +5 | +20 | 75 | 100 |

→ 積極 LTW 玩家後期傷害明顯更高（B加成差距拉大），但當前持有也影響即時輸出（A加成）。

## 四、利息（波末直接加金幣）

```
於 pre_wave 開始時執行：
bonus_gold = min(floor(currentGold × interestRate), interestCap)
player.gold += bonus_gold
```

**關鍵設計：**
- **直接加 gold**，不改 income，不影響永久收益計算
- 有 **cap**（上限）防止後期爆炸（例如每波最多 +40g）
- 只觸發一次（無論場上有幾座純無塔）
- 計算時機：戰鬥結束 → 進入 pre_wave 時

**取捨效果：**
- 「存多一點錢，每波自然多拿一點回報」
- 但同時：存太多 = 沒有升塔 = 防守壓力

**與 A加成（當前持有）的協同：**
利息讓存錢有雙重回報：傷害高（A加成）+ 每波多拿錢（利息），進一步強化「持有金幣」的誘因，但防守壓力限制了它。

## 五、LV4 → LV5 → LV6 技能解鎖

| 等級 | 解鎖技能 | 設計意圖 |
|------|---------|---------|
| LV4 | killGold（小倍率）| 建立「殺敵賺錢」的習慣，感受初步收益 |
| LV5 | killGold（中）+ 財富積累A（當前持有）| 引入「金幣↔傷害」的即時張力，玩家開始在意存金 |
| LV6 | killGold（大）+ 財富積累A + 財富積累B（累積）+ 利息 | 完整經濟引擎，複利全開 |

**LV6 的質變感：**
- 財富積累B 在 W15+ 差距顯著（積極 LTW 玩家的獎勵）
- 利息讓每波被動獲利，存錢的玩家感覺「金幣在幫我工作」

## 六、完整協同圖

```
killGold ─→ 增加 currentGold ─→ 財富積累A 提高傷害
                │                    │
                │                    └─→ 殺敵更快 → 更多 killGold
                │
                └─→ 增加 totalEarned ─→ 財富積累B 提高傷害（持久）

利息 ────→ 每波 +bonus gold ─→ 增加 currentGold（財富積累A 的原料）
                               └─→ 增加 totalEarned（財富積累B 的原料）

LTW 送兵 → 提高 income → 每波 totalEarned 增加更多 → 財富積累B 更快增長
```

## 七、「死」的條件說明

用戶說「錢剩太多就會死」→ 分析這個機制是否確實成立：

**為什麼不能純存錢：**
1. 不花錢升塔 = 塔等級低 = 射程/攻速/傷害基礎值低
2. 財富積累加成只是補充，無法彌補不升塔的差距
3. 後期波次壓力大，不升塔的玩家會在 W12+ 被打穿

**為什麼不能花光錢：**
1. 財富積累A（當前持有）提供的傷害加成瞬間消失
2. 利息下波收益歸零
3. 若緊接著是 Boss 波，傷害暫時低落

→ **最優解在兩端之間**：升完塔後盡量保留一定金幣，讓 A 加成維持在一個合理水位。
