# 概念細化 8：財富積累A + 利息的實作邊界設計

## 確認技能組合

純無塔最終三技能：
1. **killGold** — 擊殺敵人賺額外金幣
2. **財富積累A** — 傷害隨當前持有金幣縮放（戰鬥中即時）
3. **利息** — 每波結束時，依當前金幣直接加一筆金幣

---

## 核心問題：賣塔套利

### 假設情境

波次結束 → 玩家進入 pre_wave 階段 → 賣掉所有塔 → 手持大量金幣 → 觸發利息（拿到高額利息）→ 再把塔買回來

### 數學驗證：這在任何利率下都賠錢

```
設：塔造價 250g，賣出 80% = 200g（損失 50g）
   手持 200g + 原有 100g = 300g
   利息 5%：+15g
   買回塔需 250g

結算：原有 100g + 賣塔 200g + 利息 15g = 315g
     買回塔：-250g
     剩餘：65g（原本有 100g + 一座塔）
     淨損：35g + 少了一座塔
```

利息需 > 20%（賣塔損失比）才能回本，設計上不可能給到這麼高。
→ **數學上賣塔套利永遠虧損，不是真正的 exploit。**

但這不代表完全沒問題，需要確認的是**時機問題**，不是數學問題。

---

## 真正需要解決的：觸發時機

### 問題

pre_wave 期間玩家可以賣塔、建塔、升塔。
如果利息在 pre_wave **結束前**（下一波即將開始時）才觸發：
- 玩家可以先賣塔 → 手持高額金幣 → 利息基於這筆高金幣計算 → 再用賺到的利息貼補部分損失
- 雖然數學上仍是虧損，但玩家**感覺**像在操作利息，造成遊戲體驗的怪異感

### 解法：快照機制（Snapshot）

**利息的觸發基準 = 波次戰鬥結束那一瞬間的金幣數**，不是 pre_wave 期間的金幣數。

```
時序：
戰鬥結束 → 快照 waveEndGold = g.gold → 進入 pre_wave
                                              │
                                              ↓
                      （玩家可自由賣塔建塔，金幣自由變動）
                                              │
pre_wave 結束，下波開始前：                   ↓
    bonus = floor(waveEndGold × rate)
    g.gold += bonus
    （此時才實際加金幣，但基準是快照值）
```

**效果：**
- 玩家賣塔改變不了快照值，利息固定
- 利息在 pre_wave 最末才到帳，玩家看到「+X金」的提示時正好是下波開始前
- 自然、乾淨，沒有操作空間

---

## 財富積累A 的觸發時機

財富積累A 影響的是**戰鬥中的即時傷害**，不是波末結算，觸發邏輯不同。

```
戰鬥中每次攻擊：
effectiveDmg = baseDmg + floor(g.gold / divisorA)
```

這裡讀的是**即時 g.gold**（戰鬥中持有的金幣）。

**賣塔對財富積累A 的影響：**
| 時機 | 效果 | 是否合理 |
|------|------|---------|
| pre_wave 期間賣塔 → 下波開始時 g.gold 高 | 財富積累A 加成高，但少了一座塔 | ✅ 有取捨，合理 |
| 戰鬥中賣塔 → 即時 g.gold 增加 | 無塔傷害立即提升，但現場防禦削弱 | ✅ 有取捨，合理 |

→ 財富積累A 不需要快照，讀即時值反而更有策略感。

---

## 設計邊界總結

| 機制 | 觸發時機 | 讀取值 | 玩家能操作嗎 |
|------|---------|--------|------------|
| killGold | 擊殺瞬間 | 即時 | 否（自動觸發）|
| 財富積累A | 每次攻擊 | 即時 g.gold | ✅ 賣塔可拉高（有取捨）|
| 利息 | pre_wave 末（下波前）| **快照** waveEndGold | ❌ 快照後操作無效 |

---

## 完整波次流程（加入利息）

```
1. 波次開始（startWave）
2. 戰鬥進行中
   └─ 無塔攻擊：effectiveDmg = baseDmg + floor(g.gold / divisorA)
   └─ 無塔擊殺：g.gold += killGoldBonus
3. 波次戰鬥結束（所有敵人消滅 or 逃走）
   └─ waveEndGold = g.gold  ← 快照
4. 進入 pre_wave
   └─ 玩家自由賣塔、建塔、升塔（g.gold 自由變動）
5. pre_wave 倒數結束，下波即將開始
   └─ bonus = min(floor(waveEndGold × rate), cap)
   └─ g.gold += bonus
   └─ UI 顯示：「💰 利息 +X」
6. 下波開始
```

## 技能參數草案

**財富積累A：**
```js
makeSkill('wealthScale', {
  divisor: 20,   // 每 20g 當前持有 = +1 傷害
  cap: 50        // 最多 +50 傷害（需 1000g 才封頂，設計上很難達到）
})
```

**利息：**
```js
makeSkill('interest', {
  rate: 0.05,    // 5%
  cap: 40        // 每波最多 +40g（防後期爆炸）
})
```

**效果預覽：**
| currentGold（快照）| 利息（5%，cap 40）| 財富積累A（+傷害）|
|------------------|-----------------|-----------------|
| 100g | +5g | +5 |
| 400g | +20g | +20 |
| 800g | +40g（封頂）| +40 |
| 1200g | +40g（封頂）| +50（封頂）|

---

## 實作需新增的欄位

| 欄位 | 位置 | 用途 |
|------|------|------|
| `g.waveEndGold` | game.js | 波次結束時快照，利息基準 |
| 利息結算邏輯 | startWave 前 or pre_wave 結束 | 讀 waveEndGold 加金幣 |
| wealthScale 技能邏輯 | doDmg() 中 | 讀即時 g.gold 加入傷害計算 |
| UI：利息提示 | pre_wave 通知區 | 顯示「💰 利息 +X」|

## 待確認

1. **divisor 值**：20（每 20g → +1 dmg）是否合理？需比對其他純塔 LV5 的傷害值
2. **利息 cap**：40g/波 是否太低或太高？（W20 有 20 波，總利息上限約 800g）
3. **快照時機**：是「最後一個敵人死亡時」還是「波次正式結束計算時」？建議後者較乾淨
4. **wealthScale 是 LV5 解鎖還是 LV4 就有（小值）**？
