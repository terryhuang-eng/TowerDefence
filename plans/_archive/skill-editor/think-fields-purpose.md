## 三個欄位的用途

### score_adj — 整塔技能預算比例
**問題**：這個塔的技能設計，想讓它比「同等級標準」多用幾%的技能預算？

```
dpsScore = LEVEL_SCORE_STD[lv] × score_adj  -  Σ技能分

adj < 1.0 → 技能「輕」，DPS 空間大（DPS 型塔）
adj = 1.0 → 標準分配
adj > 1.0 → 技能「重」，utility 價值高（如金幣/光環型塔）
```

**什麼時候動它**：
當你看分數面板的 DPS 剩餘顯示不合理時：
- `DPS 剩餘 = -XX ⚠️` → 技能超出預算 → 考慮提高 adj（承認技能更重要）或減弱技能
- `DPS 剩餘 = 非常大` → 技能太輕 → 考慮降低 adj（期待 DPS 補上）

**誰應該有非1.0的值**：
- 強力 combo 塔（如 burn+detonate 組合）→ adj 略低（技能做的事多，DPS 少一點）
- 純輔助/光環塔 → adj 略高（DPS 低是故意的）
- 大多數普通塔 → adj 保持 1.0

### scoreTarget（目標分數覆蓋）
**幾乎不需要動它**。
唯一使用情境：某塔有設計師定義的特殊分數預算，不想跟等級標準掛鉤。

留 0 = 自動計算 `STD × adj`。

### scoreWeight — 個別技能公式補償
**問題**：公式說 warp(dur=1.0) = 35pts，但實際覺得只值 20pts，用 scoreWeight 補償。

```
scoreWeight = 期望分數 / 公式分數
```

**什麼時候動它**：
- 技能效果非線性（定身 1s 和 2s 的實際差距不是 2 倍）
- 技能有特殊 combo 效果（公式沒算到）
- 平衡感覺和公式有落差時

**大多數技能不需要動**，保持 1.0 信任公式。

## 實際工作流程

```
1. 開啟 skill-editor → 切到 Towers
2. 選一個塔 → 看分數面板
3. 判斷 DPS 剩餘是否合理：
   - 正數且適中 → 不動
   - 負數 → 技能太重，考慮調 score_adj 上移或降技能參數
   - 非常大正數 → 塔可能設計意圖是「DPS 型」→ adj 降低更能體現這個
4. 如果特定技能的分數感覺不對 → 調 scoreWeight
5. 幾乎不需要動 scoreTarget
```
