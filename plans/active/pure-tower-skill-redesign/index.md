# 設計分析：純屬塔技能重新對應

## 設計哲學確認
- **純屬塔** = 獨立輸出身份，不依賴隊友，不以控制/輔助為主
- **混合塔** = 控制 / 輔助 / 特殊效果，但直接傷害較低
- 技能與塔的對應是**可調整的**，先確認每個技能的定位，再決定歸屬

---

## 一、Burn + Ignite 合併可行性

### 問題
ignite 無法獨立存在（需要 burn 在同一塔），作為獨立技能欄位不合理。

### 結論：合併為一個技能
將 `burn` 內建引燃行為：覆蓋灼燒時**自動**造成 `flat × baseDmg` 附加傷害（參數化 ignite flat 值）。
`ignite` 技能欄位可廢除，改為 burn 的進階參數 `igniteMult`。

**可行，建議合併。**

---

## 二、Detonate 定位

### 現況問題
detonate 依賴 burnStacks（需 burn 在其他塔配合或自己堆疊），獨立性弱。

### 結論：Lv6 純火專屬
detonate 作為純火 Lv6 的 signature 技能：最終形態才能觸發爆炸。
混合或低等級火塔不帶 detonate，純火塔的路線終點感強。

**可行。**

---

| detonate | ⚠️ | 依賴 burnStacks，Lv6 專屬可接受 |
| chain | ✅ | 完全獨立 |
| hpPct | ✅ | 完全獨立 |
| execute | ✅ | 獨立（HP 條件是目標狀態，非技能依賴）|
| pierce | ✅ | 獨立 |
| multishot | ✅ | 獨立 |
| ramp | ✅ | 獨立 |
| frostbite | ⚠️ | 獨立運作，但與 chill 主題重疊，功能近似 hpPct |

## 五、Chain vs Multishot vs Pierce：三者差異分析

### 現況
三者都能「一次攻擊涉及多個目標」，但機制不同：

| | chain | multishot | pierce |
|--|-------|-----------|--------|
| 目標選擇 | 射程內其他敵人（自動跳躍）| 射程內任意目標（各射一發）| 射程內所有目標（線型順序）|
| 傷害模式 | 遞減（×decay^i）| 全額（各自完整傷害）| 遞增（+dmgUp/體）|
| 技能觸發 | 每跳獨立觸發技能 | 每發獨立觸發技能 | 每體觸發技能 |
| 策略感 | 清群（跳躍擴散）| 爆發（一瞬間多打）| 穿線（獎勵密集排列）|

**不重複，各有策略意義。**

- chain = **電弧型清群**：傷害遞減，靠數量取勝，雷系最合適
- multishot = **爆發型集火**：多發同時，傷害不衰減，適合高 DPS 場景
- pierce = **穿透型獎懲**：越多敵人排成一線，後排越痛，強迫玩家考慮位置

### 三者是否都保留？
- chain ✅：雷系獨特身份，保留
- pierce ✅：穿透概念有策略深度，但「線型」的觸發規則需要確認（目前是射程圓圈內排序，而非真實路徑穿透）
- multishot ⚠️：爆發感強，但若風系已有 ramp（越打越快），multishot 的角色是否重複需考量

## 六、初步六屬身份方向（草案）

| 元素 | 純屬傷害身份 | 關鍵技能 |
|------|------------|---------|
| 🔥 火 | 燃燒累積 → 爆炸 | burn（含ignite）/ detonate(Lv6) |
| 💧 水 | **%HP 侵蝕**（待確認）| hpPct ← 水的侵蝕感 |
| ⛰️ 土 | **重擊 / 穿甲**（待確認）| pierce ← 石頭穿透？ or 純高傷害 |
| 🌪️ 風 | 高頻 → 倍率遞增 | ramp / multishot |
| ⚡ 雷 | 連鎖擴散 | chain |
| ⬜ 無 | 混沌極值 | unstable |

## 七、待確認問題

1. **水 vs 土 的 %HP 歸屬**：hpPct 侵蝕感適合水，但若土要有傷害身份，土可能需要另一個機制
2. **pierce 的線型規則**：目前是射程圓圈排序，改成「真實路徑穿透（直線）」會更有策略感但實作複雜
3. **multishot 去留**：若風系以 ramp 為核心，multishot 是補充還是冗餘？
4. **execute 歸屬**：純混合都合理，取決於哪條路線要有「收割」定位
5. **frostbite 處理**：確認移除或並入水系 %HP 語意下？
